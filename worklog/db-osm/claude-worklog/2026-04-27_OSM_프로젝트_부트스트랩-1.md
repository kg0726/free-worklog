### 구현 내용

**왜 이 단계가 필요한가:** 본격적인 적재 코드(C3~C6) 작성 전에 코드가 동작할 환경(디렉토리 구조, 외부 라이브러리, 환경변수 템플릿, git 추적 정책)을 먼저 갖춰두는 단계. 이 단계가 끝나면 다음 커밋부터는 의존성 추가 고민 없이 로직에만 집중할 수 있다.

**왜 자율/data/osm-loader/에 두었나:** 작업지시 1번이 "새 프로젝트"라고 표현했고, be(Spring Boot) 빌드 산출물·gitignore 정책과 분리하기 위해 형제 디렉토리에 둠. 자율/이 git 루트라 별도 git init은 하지 않음.

**왜 venv를 프로젝트 폴더 안에 두었나:** 시스템 Python 전역에 설치하면 다른 프로젝트와 의존성이 충돌할 수 있음. 프로젝트 폴더 안에 격리된 venv를 만들어 이 프로젝트 전용 환경을 분리함. `.gitignore`에 `venv/`를 넣어 git 추적에서 제외.

**왜 `.env`는 자율/.gitignore에 맡겼나:** 자율/.gitignore가 이미 `*.env`/`*.env.*`/`!*.env.example` 정책을 처리하고 있어, 같은 규칙을 osm-loader/.gitignore에 중복 정의할 필요가 없음. Python 관련(`__pycache__/`, `*.pyc`, `venv/`, `.python-version`)만 osm-loader/.gitignore에 추가함.

**왜 `__init__.py`가 빈 파일로 필요한가:** Python이 `loader/`를 패키지로 인식하게 만드는 표식. 이게 있어야 `python -m loader.main` 같은 모듈 단위 실행이 가능해짐.

### 설정 파일 / 환경변수

#### `requirements.txt`
| 패키지 | 버전 제약 | 용도 |
|--------|----------|------|
| `psycopg2-binary` | `>=2.9` | PostgreSQL 드라이버. `-binary`는 사전 컴파일된 바이너리 사용(컴파일 도구 불필요) |
| `requests` | `>=2.31` | Overpass API HTTP 호출용 |
| `python-dotenv` | `>=1.0` | `.env` 파일에서 환경변수 로드 |

설치 결과 (venv 안): psycopg2-binary 2.9.12, requests 2.33.1, python-dotenv 1.2.2.

#### `.env.example` (환경변수 템플릿)
| 키 | 기본값 | 설명 |
|----|--------|------|
| `POSTGRES_HOST` | `localhost` | DB 호스트. 운영 적재 시 원격 IP로만 변경 |
| `POSTGRES_PORT` | `5432` | DB 포트 |
| `POSTGRES_DB` | `running` | DB 이름 |
| `POSTGRES_USER` | `root` | DB 유저 |
| `POSTGRES_PASSWORD` | `ssafy1234!` | DB 비밀번호 (로컬 개발용) |

`.env` 파일은 `cp .env.example .env`로 사본을 만들어 사용. 자율/.gitignore가 `.env`를 ignore.

```

venv/
.python-version
```
환경변수 관련은 자율/.gitignore에 위임.

### 확인된 이슈 / 결정 사항
- **Python 버전 확인**: 시스템에 Python 3.11.9 설치 확인 (`py --version`). 작업지시 요구사항 3.11+ 충족.
- **dotenv 버전 호환성 메모**: `python-dotenv 1.2.2`는 `__version__` 어트리뷰트가 없음. import 자체는 정상이므로 적재 로직에서 `from dotenv import load_dotenv` 형태로 사용하면 영향 없음.
- **pip 업그레이드**: venv 안 pip 24.0 → 26.1로 업그레이드. 의존성 설치 안정성 목적.
